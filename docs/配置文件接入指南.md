### 接入（自定义配置函数）步骤总结

核心是使用 `@nestjs/config` 加载一个自定义函数，该函数返回嵌套的配置对象，从而实现环境变量的组织。

#### 1. 安装依赖库

```bash
npm install @nestjs/config
```

`@nestjs/config` 内部依赖 `dotenv`，会自动读取项目根目录下的 `.env` 文件。

#### 2. 准备环境变量文件（可选）

在项目根目录创建 `.env` 文件，定义扁平的环境变量：

```
PORT=3000
DATABASE_HOST=localhost
DATABASE_PORT=5432
DATABASE_USER=admin
DATABASE_PASSWORD=secret
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
```

#### 3. 创建自定义配置文件

新建 `config/configuration.ts`，导出一个函数，返回嵌套对象：

```typescript
// config/configuration.ts
export default () => ({
  port: parseInt(process.env.PORT, 10) || 3000,
  database: {
    host: process.env.DATABASE_HOST || 'localhost',
    port: parseInt(process.env.DATABASE_PORT, 10) || 5432,
    username: process.env.DATABASE_USER,
    password: process.env.DATABASE_PASSWORD,
  },
  redis: {
    host: process.env.REDIS_HOST,
    port: parseInt(process.env.REDIS_PORT, 10),
  },
});
```

#### 4. 在根模块中导入 ConfigModule 并加载配置

在 `AppModule`（或任何需要全局使用的模块）中导入 `ConfigModule`，并通过 `load` 选项加载自定义配置函数：

```typescript
// app.module.ts
import { Module } from '@nestjs/common';
import { ConfigModule } from '@nestjs/config';
import configuration from './config/configuration';

@Module({
  imports: [
    ConfigModule.forRoot({
      load: [configuration], // 加载自定义配置
      isGlobal: true, // 可选，设为全局后无需在每个模块重复导入
    }),
  ],
})
export class AppModule {}
```

#### 5. 在服务中使用 ConfigService 获取嵌套配置

在需要使用配置的 provider 中注入 `ConfigService`，然后通过点语法访问嵌套属性：

```typescript
// app.service.ts
import { Injectable } from '@nestjs/common';
import { ConfigService } from '@nestjs/config';

@Injectable()
export class AppService {
  constructor(private configService: ConfigService) {}

  getDatabaseHost() {
    // 获取嵌套属性：database.host
    return this.configService.get<string>('database.host');
  }

  getRedisConfig() {
    // 获取整个 redis 对象
    return this.configService.get('redis');
  }
}
```

#### 6. （可选）类型提示增强

为了获得更好的类型支持，可以定义接口并断言返回值：

```typescript
interface DatabaseConfig {
  host: string;
  port: number;
  username: string;
  password: string;
}

// 使用时
const db = this.configService.get<DatabaseConfig>('database');
```

---

### 关键点说明

- **嵌套映射**：环境变量采用 `DATABASE_HOST` 命名，在配置函数中手动组装成 `{ database: { host } }`。
- **默认值**：通过 `||` 或 `parseInt` 提供默认值，避免运行时缺少变量。
- **ConfigModule 作用域**：`isGlobal: true` 可让 `ConfigService` 在所有模块中直接使用，无需重复导入。

通过以上步骤，即可在 NestJS 中实现环境变量的嵌套组织与管理。
