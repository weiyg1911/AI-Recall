# 代码质量工具使用指南

本项目已配置完整的代码质量工具链，确保代码质量和团队协作效率。

## 📋 工具概览

### 1. ESLint - 代码检查

自动检查代码质量问题和不规范的代码模式。

### 2. Prettier - 代码格式化

统一的代码格式化工具，确保代码风格一致。

### 3. Husky - Git Hooks

在 Git 操作时自动执行脚本，确保代码质量。

### 4. lint-staged - 增量检查

只对暂存区（staged）的文件运行检查，提高性能。

### 5. commitlint - 提交信息规范

强制执行统一的提交信息格式。

## 🚀 使用方式

### 日常开发

#### 自动运行（推荐）

配置了 Git hooks，以下操作会自动执行：

**提交代码时：**

```bash
git add .
git commit -m "feat: 添加新功能"
# 自动执行：
# 1. lint-staged 对暂存文件运行 ESLint 和 Prettier
# 2. commitlint 检查提交信息格式
```

#### 手动运行

**检查所有文件：**

```bash
pnpm lint         # 运行 ESLint
pnpm lint:fix     # 自动修复问题
```

**格式化所有文件：**

```bash
pnpm format
```

## 📝 提交信息规范

### 格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

### Type ��型

- `feat` - 新功能
- `fix` - 修复 Bug
- `docs` - 文档更新
- `style` - 代码格式（不影响功能）
- `refactor` - 重构
- `perf` - 性能优化
- `test` - 测试相关
- `build` - 构建系统
- `ci` - CI 配置
- `chore` - 其他更改

### 示例

**简单提交：**

```bash

git commit -m "fix: 修复验证码发送失败的问题"
```

## 🐛 常见问题

### Q: 提交时被 hook 拦住怎么办？

```bash
# 查看 lint 错误
pnpm lint

# 自动修复问题
pnpm lint:fix

# 然后重新提交
git add .
git commit -m "..."
```

### Q: 如何跳过 hooks（不推荐）？

```bash
git commit --no-verify -m "..."
```

### Q: 提交信息格式错误？

```bash
# 正确格式
git commit -m "feat: 描述"        # ✅
git commit -m "feat：描述"        # ❌ 使用中文冒号
git commit -m "feat 描述"         # ❌ 缺少冒号
```
